# 小本地模型：不要把"上一轮回复原文"塞进提示

## 失败信号

- 连续几轮 3B / 7B 输出高度雷同：开场同一句、同段人物/地点清单重复出现，
  结尾追问几乎一致。
- 玩家选择不同（align / challenge），文本差异极小。
- 提示里出现 `[[PREV_REPLY_1]]…[[/PREV_REPLY_1]]`、`历史轮 N:`、`---` 分隔
  或任何以"给你参考"的形式把上一轮正文喂回去的写法。

## 根因

3B 级别本地模型会把提示中出现的大段"上一轮正文"视为模板/续写对象，而不是
"避免项"。即使加上"禁止复述"的说明文字，也会因为上下文里已经有了成型
段落而大概率沿用其句式/词表/开场。

## 验证过的修复

1. **不要**在提示里放上一轮的完整正文或摘要段落。
2. 仅抽取"指纹"作为负样本列表：
   - 头 20–24 字（作为"禁开场"）；
   - 尾 24–28 字（作为"禁结尾/追问句式"）；
   - 中段一两个整句（`/[。！？?!]([^。！？?!]{6,40})[。！？?!]/`）。
   以"禁句 N：…"形式列出即可，并明确说明"换新的切入点 / 新的比喻"。
3. 在系统提示里把敏感格式（`---`、`===`、`[[PREV_REPLY`、`历史轮`、markdown
   标题）列入"禁止输出"清单。
4. 引擎侧把 `repeatPenalty` 提到 `{ penalty: ~1.22, lastTokens: 640,
   frequencyPenalty: 0.5+, presencePenalty: 0.5+ }`，`temperature` ~0.88，
   `topP` ~0.92，从采样层再压一层重复。
5. 属性状态（例如傲娇度/好感度）必须在系统提示里以"## 本轮态度即时校准
   (score=X)"这种单独段落出现，明确"贯穿每一段，不许只在首段生效"。按分数
   分档（-3…+3）给出具体行为描述，而不是笼统的"根据态度调整"。

## 反例（已废弃）

- 把上一轮整段塞进 `# 最近三轮回顾` 再加一行"不要复述"。
- 只把禁用话题以 topic-id 形式给出（模型读不懂 id，仍会提到 `盘古/女娲`）。
  应同时列出该话题关联的具体人物/典籍专名。

## 适用范围

Qwen2.5-3B / 7B Instruct GGUF + node-llama-cpp ChatML session，ContextSize
8k 以内。更大的模型（>= 14B）对"参考不复述"指令遵循较好，可以酌情放宽。
