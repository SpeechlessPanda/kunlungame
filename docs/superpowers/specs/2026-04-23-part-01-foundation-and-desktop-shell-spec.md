# Part 01 Spec: 基础工程与桌面壳

## 1. 目标

建立可持续开发的 Windows 桌面项目基座，为后续所有部分提供稳定的运行容器、构建链路、测试入口和最小安全边界。

## 2. 范围

本部分只负责以下内容：

1. Electron + Vue 3 + TypeScript 项目骨架。
2. Electron 主进程、预加载层、渲染层的最小职责划分。
3. 开发、构建、测试、覆盖率的基础配置。
4. 基础目录结构与命名约定。
5. 最小应用窗口、最小页面挂载和最小 IPC 暴露边界。

## 3. 明确不做

1. 不实现主线推进逻辑。
2. 不实现知识库、检索和 AI 请求。
3. 不实现正式 UI 视觉设计。
4. 不接入真实素材文件。

## 4. 依赖与前置

1. 依赖已确认技术栈：Electron、Vue 3、TypeScript、Vite、Vitest、Playwright。
2. 默认运行平台为 Windows。
3. 本部分是所有后续部分的前置依赖。

## 5. 契约与边界

1. 主进程只负责窗口、文件系统、应用生命周期和受控 IPC。
2. 渲染层不得直接读写本地文件系统。
3. 预加载层只暴露白名单 API，不直接透出 Node 任意能力。
4. 共享类型放在 shared 层，避免渲染层和主进程重复定义。

## 6. 关键流程

1. 应用启动。
2. Electron 创建窗口。
3. 预加载脚本注入受控 API。
4. Vue 应用挂载成功。
5. 测试与构建命令可运行。

## 7. 错误处理要求

1. 启动失败时要能在控制台输出明确错误。
2. 缺失配置时要采用可解释的默认值或清晰报错。
3. IPC 未注册调用时，渲染层不能崩溃整个应用。

## 8. 测试策略

黑盒测试：

1. 应用窗口可启动。
2. 渲染层能正常挂载。
3. 最小页面元素可见。

白盒测试：

1. 预加载层只暴露预期 API。
2. 环境配置解析逻辑正确。
3. 主进程窗口配置值符合预期。

## 9. 验收条件

1. `pnpm install` 成功。
2. `pnpm dev` 可打开桌面窗口。
3. `pnpm build` 成功。
4. `vitest` 与 `playwright` 具备可执行入口。
5. 工程目录、命名、职责边界与计划书一致。

## 10. 交付物

1. 基础项目脚手架。
2. 构建与测试配置。
3. 最小应用窗口与挂载页。
4. 基础 README 或运行说明。

## 11. 执行状态

已完成：

1. 已建立 Electron + Vue 3 + TypeScript + Vite 的最小桌面壳骨架。
2. 已建立主进程、预加载层、渲染层的最小职责划分，并通过 shared 类型层复用桌面桥接契约。
3. 已把模型启动规划器接入桌面壳，主进程可产出 `desktop:get-startup-snapshot` 供后续设置页与启动流程消费。
4. 已提供 `pnpm dev`、`pnpm build`、`pnpm typecheck`、`pnpm test`、`pnpm test:e2e` 基础入口。
5. 已完成白盒验证：桌面桥白名单 API、主窗口安全配置、启动摘要适配、启动失败日志前缀。
6. 已完成编译验证：Node/Electron 类型检查、Vue 类型检查、electron-vite 构建。

明确延期：

1. 黑盒桌面窗口启动与 Playwright 场景脚本暂未落具体用例；当前仅保留执行入口，待后续正式 UI 与交互壳细化后补齐。
2. 当前渲染层仍是最小挂载页，只承担桌面壳可运行性验证，不承载正式视觉设计与故事界面。

## 12. 风险与回滚边界

1. 若桌面脚手架选择错误，后续所有 spec 都会受影响，因此本部分一旦验收通过，不应再轻易更换框架。
2. 若 IPC 边界不收紧，后续功能会把桌面壳和业务层耦合在一起。
