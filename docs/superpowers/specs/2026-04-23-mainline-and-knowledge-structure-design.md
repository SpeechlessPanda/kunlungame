# 主线与知识库结构化设计

## 1. 目标

基于现有的 [docs/knowledge-base/cultural-knowledge.md](docs/knowledge-base/cultural-knowledge.md) 设计一条面向中国用户、全程中文输出、单主线、可稳定支撑 5 分钟以上体验的叙事主线，并将原始知识库重构为可编译、可检索、可测试、可约束剧透边界的结构化内容层。

本设计同时定义该主线对 Part 02、Part 03、Part 04、Part 05 的回填要求，为后续非 UI 实现提供统一边界。

## 2. 已确认约束

1. 游戏目标用户为中国用户，模型必须以中文回答。
2. 玩家只有两个选项，选项只影响 AI 风格，不影响主线事实或节点走向。
3. 主线必须单链路推进，不做分支剧情。
4. 主线要覆盖从昆仑神话到当代文化自觉的纵向文明脉络。
5. 玩家身份为现代中国青年，但在文化认知上像“失忆者”，需要逐步重拾文化记忆与文化自信。
6. AI 身份不是讲义式老师，而是亲切、清楚、具陪伴感的文化向导。
7. 整体语气以现代口语为基础，在关键节点允许抒情升格。
8. Part 06 跳过；Part 07 仅继续非 UI 的资源位、模式与契约部分。

## 3. 设计结论

采用“A+C 混合方案”：

1. 外层结构使用稳定的“文明回廊”单主线，保证节点推进、检索过滤、提示词边界、测试设计都可控。
2. 内层动机使用“失落文脉修复”，让玩家不是旁观历史，而是在一段段文明记忆中重新认出“这与我有关”。
3. 每个节点都回答一个逐层递进的问题：
   - 文明从哪里开始想象自己？
   - 中国人怎样定义秩序与意义？
   - 文脉如何扩展、融合、再创造？
   - 文脉如何在断裂中幸存？
   - 为什么今天的我仍然需要它？
4. 终点不是“完成答题”，而是玩家完成从“知道一些传统文化”到“承认这些文化与我有关，并愿意继续传下去”的转换。

## 4. 主线概念

主线名暂定为“昆仑回声”。

开场时，玩家处于一种对本民族文化“熟悉又陌生”的状态，只记得零碎符号，例如昆仑、长安、诗词、故宫，却无法把它们连成完整脉络。文化向导发现玩家的文化记忆断裂，于是带玩家进入一条从昆仑延展到当代的文明回廊。玩家每穿过一段回廊，不是改变历史，而是修复一段本已属于自己的文化记忆。

这条主线的情绪曲线分为三段：

1. 上升段：从神话到先秦，建立“文明如何认识自己”的根。
2. 展开段：从秦汉到清代，展示文明如何统一、开放、精致、融合、再创造。
3. 回收段：从近代断裂进入当代回响，让前面的知识重新照回现实生活，完成文化自觉的闭环。

## 5. 主线节点设计

主线固定为 8 个 story nodes，形成不可分叉链。

### 5.1 Node 01 昆仑初问

1. 作用：建立玩家“文化失忆者”的代入前提，抛出昆仑作为世界中心、文明原点、精神坐标的总问题。
2. 必经事实：昆仑的世界中心象征、西王母与昆仑的文化演化、昆仑作为天人之轴的意义。
3. 情感目标：把“陌生的传统文化”转成“我愿意继续听下去”。

### 5.2 Node 02 神话开天

1. 作用：展示中国文明如何通过神话解释天地、人类、救世与秩序起源。
2. 必经事实：盘古开天、女娲造人与补天、夸父逐日、后羿嫦娥、大禹治水。
3. 情感目标：让玩家理解中国神话不是零散传说，而是文明自我想象的起点。

### 5.3 Node 03 三皇五帝与文明起源

1. 作用：把神话过渡到文明发端，解释火、农、医、礼、历法、统一观念如何形成。
2. 必经事实：燧人、伏羲、神农、黄帝、尧舜禹、炎黄概念、多元一体的原型。
3. 情感目标：让玩家感到“炎黄子孙”不是空口号，而是文明叙事的共同身份。

### 5.4 Node 04 礼乐与诸子

1. 作用：解释中国文化如何思考秩序、伦理、变易、争鸣与意义。
2. 必经事实：周公制礼作乐、周易三层结构、绝地天通、诸子百家、稷下学宫。
3. 情感目标：让玩家理解中国文化并非单一声音，而是在争论中形成稳定核心。

### 5.5 Node 05 周秦汉唐的大一统与开放

1. 作用：展示制度、书写、史学、丝路、诗歌与都城记忆如何塑造“中国”的共同想象。
2. 必经事实：秦统一、焚书的代价、汉武帝与丝路、司马迁、唐诗、长安的开放气象。
3. 情感目标：让玩家看到文明不仅能聚合，也能向外交流并产生自信。

### 5.6 Node 06 宋元明清的精致、融合与再创造

1. 作用：表现文明进入更成熟的日常审美、技术创造、跨文化融合和文学高峰阶段。
2. 必经事实：宋词与理学、苏轼、李清照、沈括、元曲与青花、郑和、《永乐大典》、曹雪芹、《红楼梦》、京剧、四库全书。
3. 情感目标：让玩家感到中国文化不仅厚重，也细腻、灵动、会不断重写自己。

### 5.7 Node 07 近代断裂与守护

1. 作用：承接危机、痛感与守护，解释文脉为何一度断裂、又如何被人守住。
2. 必经事实：鸦片战争、戊戌变法、谭嗣同、五四、鲁迅、胡适、抗战、文化浩劫、梁思成与林徽因、改革开放后的文化复苏。
3. 情感目标：让玩家理解文化自信不是天然拥有，而是从伤痛、反思与守护中重新长出来。

### 5.8 Node 08 当代回响与文化自觉

1. 作用：把全部文明记忆接回现实，让玩家看到传统文化仍在今天以新的形式活着。
2. 必经事实：费孝通文化自觉、汉服、非遗、故宫文创、河南卫视、中国诗词大会、国家宝藏、数字文化、文明互鉴。
3. 情感目标：完成“文化回到我身边”的落点，并给出继续传承的开放结尾。

## 6. 回合与时长设计

1. 每个节点至少包含 1 个固定主题回合，关键节点可扩展到 2 个回合。
2. 推荐总轮次为 10 到 12 轮：
   - Node 01 至 Node 04 各 1 轮
   - Node 05 至 Node 07 各 2 轮
   - Node 08 1 到 2 轮
3. 单轮目标输出长度按流式体验设计为 180 到 320 字中文，可根据节点重要度浮动。
4. 按正常阅读和选择节奏，首轮闭环体验可稳定达到 5 到 8 分钟。

## 7. 选项语义设计

每轮固定输出两个动态选项，但它们只影响语气、追问深浅、情绪温度，不影响主线推进。

选项语义固定映射为：

1. 附和型：表示共鸣、接受、愿意继续被带入。
2. 反驳型：表示质疑、追问、保持距离或要求更具体证据。

运行时只记录内部态度枚举，不保存按钮表面文案为业务事实。模型必须在当前节点上下文内，为这两个内部语义生成贴合语境的中文文案。

## 8. AI 说话风格约束

1. 使用现代中文表达，避免教科书腔和翻译腔。
2. 对中国用户默认使用“我们”“我们的文化”这类共同体表述，但不得空泛煽情。
3. 每轮优先使用人物、场景、细节、问题引导，而不是连续罗列概念。
4. 附和型态度上升时，允许更温柔、更共情、更抒情。
5. 反驳型态度上升时，允许更理性、更解释性、更具论证感。
6. 无论态度如何变化，都不能更改节点必经事实和节点顺序。

## 9. 知识库结构化方案

原始 [docs/knowledge-base/cultural-knowledge.md](docs/knowledge-base/cultural-knowledge.md) 继续保留为编译源，不直接作为模型输入。编译后输出两类运行时数据。

### 9.1 Story Outline 数据

Story Outline 只服务主线推进与边界约束，建议字段如下：

1. `nodeId`
2. `title`
3. `era`
4. `coreQuestion`
5. `summary`
6. `mustIncludeFacts`
7. `recommendedFigures`
8. `backgroundMode`
9. `backgroundHint`
10. `characterCueIds`
11. `allowedKnowledgeTopics`
12. `forbiddenFutureTopics`
13. `minTurns`
14. `nextNodeId`

该层是 Part 02 与 Part 04 的主数据源。

### 9.2 Knowledge Entry 数据

Knowledge Entry 只服务检索和引用，建议字段如下：

1. `entryId`
2. `era`
3. `topic`
4. `title`
5. `figures`
6. `summary`
7. `details`
8. `emotionTags`
9. `applicableNodeIds`
10. `depthLevel`
11. `quoteCandidates`
12. `sourceSection`

结构化原则如下：

1. 一个条目只表达一个足够清晰的事实单元或人物主题，不保留大段混合说明。
2. `applicableNodeIds` 用于约束哪些节点可以检索到该条目。
3. `depthLevel` 用于控制基础普及和深度补充。
4. `emotionTags` 用于帮助模型在不同节点选出更合适的情感细节。
5. `quoteCandidates` 只存适合对话引用的短句，不存大段原文堆砌。

### 9.3 结构化分层

建议将 cultural knowledge 拆为以下主题层：

1. `myth-origin`：昆仑、盘古、女娲、夸父、后羿、大禹。
2. `civilization-origin`：三皇五帝、炎黄、考古对应、文明起源争论。
3. `order-and-thought`：礼乐、周易、三坟五典、诸子百家、琴笙文化。
4. `empire-and-openness`：周秦汉唐、统一、史学、丝路、长安。
5. `fusion-and-refinement`：宋元明清的文学、科技、艺术、航海、典籍。
6. `rupture-and-guardianship`：晚清危机、五四、近现代守护者、文化劫难与复苏。
7. `contemporary-return`：文化自觉、非遗、文创、节目化传播、数字文化。
8. `dialogue-samples`：仅作为 prompt 示例源，不直接作为知识事实源。

## 10. 对 Part 02 到 Part 05 的回填要求

### 10.1 Part 02 内容合同

需要从最小 schema 升级为可表达真实主线的合同：

1. story node 要支持固定链路、节点问题、必经事实、前后节点关系。
2. 需要补充 `allowedKnowledgeTopics` 与 `forbiddenFutureTopics`，用于反剧透边界。
3. 需要将 `recommendedFigures`、`backgroundHint`、`characterCueIds` 明确为可测试字段。
4. 需要为未来 3D 主角与角色资产保留资源 cue，而不是现在就绑定真实文件路径。

### 10.2 Part 03 知识编译

需要从“把 Markdown 拆成知识段落”升级为“编译出主线数据 + 检索数据”：

1. 编译产物至少分成 story outline 和 knowledge entries 两类。
2. 检索接口要支持按 `nodeId`、`topic`、`depthLevel` 过滤。
3. 空检索时允许降级，但不能越过当前节点边界拉取未来内容。
4. 原始 knowledge markdown 中的对话示例与知识事实要分开处理。

### 10.3 Part 04 状态与存档

需要从最小推进逻辑升级为真实主线恢复逻辑：

1. 默认入口节点固定为 Node 01。
2. 节点推进只允许按 `nextNodeId` 前进。
3. `historySummary` 要改为面向“文化记忆已修复片段”的摘要方式。
4. 存档恢复时需要验证当前节点仍存在于最新 story outline 中。

### 10.4 Part 05 AI 编排与流式输出

需要让 prompt builder 和流编排严格服从主线：

1. system prompt 中必须写明单主线、中文输出、陪伴式导览、禁止越节点剧透。
2. user/context prompt 中必须带上当前节点问题、必经事实、允许引用人物、知识片段。
3. 动态选项生成必须映射到附和型与反驳型语义。
4. 流式事件层要把“文本片段”“选项生成”“完成”“失败”严格分离。

## 11. 测试与验收设计

### 11.1 内容层测试

1. story outline 节点必须形成单链路且无断链。
2. 每个节点必须存在核心问题、必经事实、下一节点或结束标记。
3. `forbiddenFutureTopics` 必须覆盖后续关键主题，防止提前剧透。

### 11.2 知识编译测试

1. cultural knowledge 能编译出稳定的 story outline 与 knowledge entries。
2. 结构化条目在 `applicableNodeIds` 上分布正确。
3. 对话示例不会污染事实检索数据。

### 11.3 状态层测试

1. 不同态度选择只改变风格值，不改变节点事实推进。
2. 中断恢复后，仍回到正确的当前节点与历史摘要。
3. story outline 更新后，旧存档若指向失效节点，必须给出明确恢复策略。

### 11.4 AI 编排测试

1. prompt 中包含中文输出、节点问题、反剧透边界。
2. 选项总是二选一，且分别映射附和型与反驳型。
3. 检索只能拿到当前节点允许的知识。
4. 在主线与模型接通后，需增加“主线 + 模型”协作测试，验证回复能围绕当前节点展开且不越界。

## 12. 非目标

1. 本设计不定义 Part 06 的 UI 布局与交互壳。
2. 本设计不在本轮绑定真实背景图、BGM 或 3D 模型文件。
3. 本设计不引入分支剧情、开放输入或多结局结构。

## 13. 风险与对策

1. 风险：知识条目拆分过粗，模型容易自由发散。
   对策：强制事实单元化，给每条 entry 明确 node 适用范围。
2. 风险：主线节点过少，体验不足；过多，首轮节奏拖沓。
   对策：先固定 8 节点，再通过 10 到 12 回合控制首轮时长。
3. 风险：近代部分若只讲苦难，结尾会失去抬升。
   对策：把“断裂”与“守护”并写，并用当代回响完成回收。
4. 风险：AI 口吻过于讲义化，破坏陪伴感。
   对策：把人物、细节、追问式表达写入 prompt 约束和测试断言。

## 14. 后续执行顺序

1. 先按本设计补齐 Part 02 到 Part 04 的真实主线适配。
2. 再进入 Part 05，实现 prompt builder、流式编排与动态双选项。
3. 跳过 Part 06。
4. 执行 Part 07 中非 UI 的资源位、背景模式、角色 cue 契约补全。
5. 在主线与模型实际接通后，再补 Part 08 的端到端验收、覆盖率复核和审计记录。
