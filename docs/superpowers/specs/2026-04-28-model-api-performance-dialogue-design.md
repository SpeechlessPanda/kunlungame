# Model API Performance Dialogue Design

## Goal

Make the game default to an OpenAI-compatible API-key model path while keeping local GGUF as an offline fallback, and improve dialogue quality, continuity, persona, option labels, and local runtime diagnostics/performance.

## Current Findings

- The local model path flows from `RuntimeState.settings.preferredModelMode` through Electron IPC into `runMainlineTurn`; remote API provider selection now bypasses local GGUF checks when an API key and model are configured.
- Option labels are deterministic and generated by `buildGalgameOptionLabels`; the mock path has older node-specific labels that still include unnatural phrasing such as plural address.
- Prompt persona now describes `昆仑子` as a culture guide and explicitly addresses a single player as `你`.
- `realLlamaSession` calls `getLlama({ gpu: 'auto' })`, filters the known benign `</s>` warning, logs backend/device/VRAM diagnostics, and uses bounded context/output defaults for speed.
- Runtime memory now passes cleaned recent model replies as compressed continuity context while still using fingerprints to avoid repetitive openings.

## Design

### Model Provider Settings

Extend runtime settings with a `modelProvider` union:

- `openai-compatible`: recommended/default. Requires an API key and uses OpenAI chat-completions streaming.
- `local`: uses existing GGUF profiles as offline fallback.

OpenAI-compatible settings include `apiKey`, `baseUrl`, and `model`. The default base URL is `https://api.openai.com/v1`; the default model is `gpt-4o-mini` for speed, Chinese fluency, and instruction following. The UI should also recommend stronger options: `gpt-4.1-mini` or `gpt-4o` for higher quality where available.

### Remote Adapter

Add a dependency-compatible adapter that accepts `StoryPrompt` and streams OpenAI-format chat completion chunks. It must:

- Send `prompt.system` as system role and `prompt.user` as user role.
- Use `stream: true`, moderate temperature, and a bounded token budget.
- Parse SSE `data:` lines without adding a heavy SDK dependency.
- Return clear errors for missing API key, HTTP failures, and malformed provider output.
- Do not silently switch to a local model after a configured remote stream has failed mid-turn; the UI should surface the remote error so already-streamed text is not mixed with a second model's fallback response. Local fallback is automatic only when API provider is selected but no API key is configured.

### Context Retention

Keep existing `recentTurns` but transform it into a compact dialogue memory section rather than only negative fingerprints. The prompt should include:

- Node progress summary from runtime state.
- The last several cleaned model replies as compressed continuity bullets.
- The latest exact reply when it is short enough, capped to avoid local model slowdown.

### Persona And Options

Rename the guide persona to `昆仑子`. The character addresses exactly one player as `你`; plural `你们` is forbidden unless quoting historical groups. Align labels should express wonder/acceptance, such as recognizing long cultural continuity. Challenge labels should question reasoning, evidence, cost, or legitimacy.

### Local Runtime Performance

Keep GPU auto-detection, but make it observable and faster:

- Use a quiet node-llama logger that suppresses the known benign `control-looking token '</s>'` warning.
- Log selected backend/device/VRAM information once when a local session starts.
- Reduce the local context target to a bounded range suitable for the current prompt and enable flash attention where supported.
- Lower default max output tokens for local responses; remote models can use a similar cap.
- Keep `KUNLUN_FORCE_CPU=1` as a debugging override.

## Tests

- Unit tests for runtime settings defaults and backwards-compatible parsing.
- Unit tests for option label tone and single-player wording.
- Unit tests for prompt persona, context memory injection, and forbidden plural/persona terms.
- Unit tests for OpenAI-compatible streaming request shape and SSE parsing.
- Mainline runner tests for choosing remote dependencies when API settings are configured and local dependencies otherwise.
- Existing full test suite plus typecheck after implementation.

## Deferred

- Secure OS credential storage for API keys. This iteration stores the key in the existing runtime save path because the current app already persists runtime settings there.
- Full remote provider marketplace support. Only OpenAI-compatible chat completions are in scope.
- Real external API smoke test, because it requires the user's actual key.
