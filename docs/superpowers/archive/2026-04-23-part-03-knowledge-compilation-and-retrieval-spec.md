# Part 03 Spec: 知识库编译与检索层

## 1. 目标

建立从 Markdown 原始内容到结构化知识数据的编译链路，并提供面向主线节点的稳定检索能力。

## 2. 范围

1. Markdown 内容解析。
2. 结构化知识 JSON 生成。
3. 基于节点、关键词、主题的检索。
4. 检索结果数量限制与排序策略。
5. 检索为空时的降级策略。

## 3. 明确不做

1. 不负责主线推进。
2. 不直接调用 AI。
3. 不负责 UI 展示。
4. 不负责真实内容的最终质量审核。

## 4. 依赖与前置

1. 依赖 Part 02 的内容契约。
2. 为 Part 05 提供检索输入。

## 5. 契约与边界

1. 输入是受约束的 Markdown 原文。
2. 输出是 `storyOutline.json` 与 `knowledgeEntries.json` 两类稳定运行时产物。
3. 检索器只关心数据匹配，不关心生成语气。
4. 排序策略优先当前 story node 显式命中，其次关键词命中。
5. 检索降级不得跨越当前节点或允许主题边界。

## 6. 关键流程

1. 读取 `docs/knowledge-base/cultural-knowledge.md`。
2. 按一级章节映射主题层，按二级小节拆分事实条目。
3. 同步写出 `storyOutline.json` 与 `knowledgeEntries.json`。
4. 在运行时按当前节点、允许主题与关键词筛选候选知识片段。
5. 对话示例章节不进入事实检索数据。

## 7. 错误处理要求

1. Markdown 格式错误时输出明确定位信息。
2. 编译结果为空时阻止继续进入正式检索流程。
3. 检索为空时返回受边界约束的降级结果，而不是抛未处理异常。

## 8. 测试策略

黑盒测试：

1. 给定 cultural knowledge 源 Markdown，能生成 story outline 与 knowledge entries 双产物。
2. 给定节点和关键词，能返回当前节点允许范围内的预期知识条目。
3. 检索为空时，返回受节点边界约束的降级结果。

白盒测试：

1. 解析器对一级章节、二级小节和忽略章节的处理正确。
2. 检索排序规则稳定。
3. 条目过滤上限、节点边界和主题边界一致。

## 9. 验收条件

1. 存在可重复执行的知识编译脚本。
2. 编译结果能被版本化管理或稳定生成。
3. 检索层能按节点与允许主题返回有限且相关的结果。
4. 编译错误与检索空结果都有清晰处理路径。
5. 已生成 `storyOutline.json` 与扩展后的 `knowledgeEntries.json`。

## 10. 交付物

1. 编译脚本。
2. 编译后的 story outline 与 knowledge files。
3. 检索服务模块。
4. 内容格式验证测试。

## 11. 执行状态

Status: completed

已完成：

1. 已建立基于 `docs/knowledge-base/cultural-knowledge.md` 与 `src/content/source/mainlineOutline.ts` 的双源编译入口。
2. 已建立可重复执行的知识编译脚本 `pnpm knowledge:compile`，稳定输出 `storyOutline.json` 与 `knowledgeEntries.json`。
3. 已将 cultural knowledge 按章节映射为结构化知识条目，并忽略对话示例章节的事实入库。
4. 已建立运行时检索模块，支持按当前节点、允许主题、关键词和主题排序，并在无候选时返回受边界约束的降级结果。
5. 已完成黑盒与白盒聚焦验证，覆盖双产物编译、节点边界检索与空结果降级。

明确延期：

1. 暂不实现复杂 Markdown 语法、富文本内嵌资源与多文档引用图谱。
2. 当前条目拆分仍以章节小节为主，后续可继续细化到更小事实单元与人物级条目。

## 12. 风险与回滚边界

1. 若 Markdown 规则过松，后续 AI 生成会不断吃到脏数据。
2. 若检索器承担过多生成职责，会和 Part 05 耦合。
3. 若节点边界过滤失效，后续 prompt 将直接产生越节点剧透风险。
